# FDCR 无攻击场景实验分析报告

**生成时间**: 2026年01月19日 20:58:22

## 一、实验目的

在**0%恶意客户端**（无攻击）的场景下，测试去作弊版FDCR的误报率（FPR）和过滤比例，验证：

1. FINCH+Fisher在异构/Non-IID场景下是否存在天然不稳定性
2. 即使没有攻击，去作弊版是否仍会产生高误报
3. 数据异构性（α值）对误报的影响

## 二、实验配置

| 配置项 | 值 |
|--------|-----|
| 攻击类型 | backdoor (恶意率=0%) |
| 恶意客户端比例 | 0% (无攻击) |
| 数据分布 | α=0.9 (接近IID), α=0.1 (高度异构) |
| 服务器版本 | OurRandomControlNoCheat (去作弊版) |
| 通信轮次 | 100 rounds |
| 总客户端数 | 10 |

## 四、关键发现

1. **数据异构性显著影响误报率**: 在高度异构场景(α=0.1)下，FPR比IID场景(α=0.9)高 3.40%

2. **大量良性客户端被误判**: 在无攻击场景下，平均有 36.5% 的良性客户端被错误过滤

3. **误报频繁发生**: 在 100.0% 的轮次中产生了误报

4. **即使在IID场景下也存在误报**: 100.0% 的轮次产生误报

## 五、结论

### 5.1 核心结论

**FINCH+Fisher在异构/Non-IID场景下存在天然的不稳定性，即使在无攻击的情况下也会产生大量误报。**

### 5.2 证据链

```
1. 作弊版在有攻击时表现优异 (TPR 99.6%)
   ↓
2. 去作弊版在有攻击时表现很差 (TPR 21.3%, FPR 38.5%)
   ↓
3. 去作弊版在无攻击时仍有高FPR (α=0.1: 36.50%)
   ↓
结论: FDCR的检测能力完全来自'作弊'路径
      Fisher信息+FINCH聚类在Non-IID下天然不稳定
```

### 5.3 对FDCR方法的质疑

1. **方法有效性存疑**: 在无攻击场景下产生高误报，说明方法本身无法有效区分客户端
2. **数据异构性敏感**: 在Non-IID场景下性能显著下降，不适用于实际联邦学习场景
3. **依赖先验知识**: 原始实现的高性能来自使用先验知识的'作弊'路径
